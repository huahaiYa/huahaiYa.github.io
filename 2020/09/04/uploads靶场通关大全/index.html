uploads靶场通关大全

0x00 前言

文件上传漏洞是web漏洞中常见漏洞之一，所以学习文件上传的各种姿势也是学习的目标，本篇博文将带领大家通关uploads靶场，练习和总结各种文件上传中的奇思妙想！

0x01 环境搭建

环境方面我使用了phpstudy集成工具进行搭建，组件是apache+mysql8.0，将uploads靶场放到www目录下，开启服务直接就可以访问了。

  2. uploads靶场

0x03 突破靶场

下面开始进入正题了

第一关

任务：上传一个webshell到服务器

1.准备一个png文件，里面写入一个一句话木马

2.直接上传png文件

下方出现了图片，表示上传成功

3.修改文件后缀为php上传失败

4.查看源码发现采用了前端js检测后缀的方法，采用burpsuite抓取数据包，之后修改后缀达到绕过的目的

5.访问文件

至此第一关结束

总结：在第一关中使用了burpsuite截断数据包达到绕过前端js文件检测后缀的效果，所以使用前端检测的方式是不靠谱的哈。

第二关

第二关同样可以使用burpsuite截断绕过的方式进行绕过

1.查看源码，进行简单的审计

if (isset($_POST['submit'])) {
    if (file_exists(UPLOAD_PATH)) {
        if (($_FILES['upload_file']['type'] == 'image/jpeg') || ($_FILES['upload_file']['type'] == 'image/png') || ($_FILES['upload_file']['type'] == 'image/gif')) {
            $temp_file = $_FILES['upload_file']['tmp_name'];
            $img_path = UPLOAD_PATH . '/' . $_FILES['upload_file']['name']            
            if (move_uploaded_file($temp_file, $img_path)) {
                $is_upload = true;
            } else {
                $msg = '上传出错！';
            }
        } else {
            $msg = '文件类型不正确，请重新上传！';
        }
    } else {
        $msg = UPLOAD_PATH.'文件夹不存在,请手工创建！';
    }
}

发现本关是对文件类型进行了检测，如果不是”image/jpeg,image/png,image/gif“就会报错。

ok，只需要修改Content-Type类型就可以了，由于我上传的时候是图片，所以默认就是image/png格式，所以成功进行了绕过

第三关

同样上传png格式图片，使用burp修改后缀后放包查看结果，发现不允许.asp,.aspx,.php,.jsp文件后缀，但是对于php服务器来说php1，PHP2,php3,等后缀文件同样可以解析成为php文件，所以只需要修改文件后缀为php1即可进行绕过

访问图片能够直接进行访问

        $file_name = trim($_FILES['upload_file']['name']);
        $file_name = deldot($file_name);//删除文件名末尾的点
        $file_ext = strrchr($file_name, '.');

        $file_ext = trim($file_ext); //收尾去空

        if (!in_array($file_ext, $deny_ext)) {

                $msg = '上传出错！';
            }
        } else {
            $msg = '此文件不允许上传!';
        }
    } else {
        $msg = UPLOAD_PATH . '文件夹不存在,请手工创建！';
    }
